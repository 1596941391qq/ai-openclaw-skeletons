# AGENTS.md 模板

## 一句话

这个仓库的主任务是：[描述你的核心业务目标]。

## 启动顺序

每个新 Agent / 新频道进入本项目后，默认按这个顺序工作：

1. 先读本文件（AGENTS.md）
2. 根据当前任务，只读必要文件
3. 先判断"已有能力能不能直接复用"
4. 能走现成链路就不要新建平行系统

## 工作规则

### 先复用，后新增

新增脚本、表、目录、流程前，先检查：

1. 本文件是否已有对应能力
2. `Packs/` 或 `Bundles/` 是否已有入口
3. `workspace/` 是否已有可复用产物

如果已有 70% 可复用链路，优先补齐，不另起炉灶。

### 只读必要文件

不要全仓扫读。按任务目标只读相关入口、脚本和产物。

### 先确认 source-of-truth

做任何修改前，先判断当前任务的事实源在哪。

## 禁止事项

- 不要绕过本文件直接拍脑袋建新链路
- 不要重复创建同类脚本、同类中间层
- 不要在不确认 source-of-truth 的情况下回写状态
- 不要为单次任务引入长期复杂抽象
- 不要全仓无差别读取

## 修改原则

- 小改优先
- 入口优先
- 兼容现有链路优先
- 文档和脚本保持一致

## 交付标准

一个合格的改动，至少要回答清楚：

1. 入口脚本是什么
2. 依赖哪些资产
3. 产物落在哪里
4. 回写到哪里
5. 谁是 source-of-truth
