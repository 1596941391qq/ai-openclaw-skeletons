## 核心概念

`AGENTS.md`（或写在 `CLAUDE.md` 中）是 AI Agent 的行为契约。它不是文档，是法律。Agent 进入项目后，这个文件决定了它：

- **能做什么** — 允许的操作范围
- **不能做什么** — 明确的禁止事项
- **怎么做** — 工作流程和优先级
- **去哪找** — 资源定位地图

## 结构模板

### 最小可用结构

```markdown
# AGENTS.md

## 一句话
[项目的核心使命，一句话说清楚]

## 启动顺序
1. 先读什么
2. 再做什么
3. 默认行为是什么

## 资产地图
- 入口脚本在哪
- 配置文件在哪
- 产物落在哪

## 工作规则
- 先复用后新增
- 只读必要文件
- 先确认 source-of-truth

## 禁止事项
- 不要做 X
- 不要做 Y
```

### 完整结构

```markdown
# AGENTS.md

## 一句话
围绕 PSEO 业务，复用现有资产完成策略→挖词→发布→证据回写→数据运营闭环。

## 启动顺序
1. 先读 `PSEO-INDEX.md`
2. 根据当前任务，只读必要文件
3. 先判断"已有能力能不能直接复用"
4. 能走现成链路就不要新建平行系统
```

## 编写原则

### 1. 一句话必须精准

```markdown
# 好的
围绕 PSEO 业务完成策略→发布→运营闭环，不重复造轮子。

# 差的
这是一个 AI 自动化项目，用于各种 SEO 相关任务。
```

### 2. 启动顺序必须可执行

每一步都是具体文件路径或具体动作，不是抽象描述。

### 3. 禁止事项必须具体

```markdown
# 好的
- 不要绕过 PSEO-INDEX.md 直接建新链路
- 不要在不确认 source-of-truth 的情况下回写状态

# 差的
- 不要做不好的事情
- 注意代码质量
```

### 4. 交付标准必须可验证

```markdown
## 交付标准
一个合格的改动必须回答：
1. 入口脚本是什么
2. 依赖哪些资产
3. 产物落在哪里
4. 回写到哪里
5. 谁是 source-of-truth
```

## 验证

- [ ] Agent 进入项目后能正确复述"一句话"
- [ ] Agent 按启动顺序加载上下文
- [ ] Agent 在禁止事项边界内工作
- [ ] 新成员读完 AGENTS.md 后能独立开始工作

## 常见问题

**Q: AGENTS.md 和 CLAUDE.md 什么关系？**
A: 可以合并写在 CLAUDE.md 中。分开写时，AGENTS.md 侧重行为契约，CLAUDE.md 侧重项目说明。

**Q: 写太长 Agent 会忽略吗？**
A: 会。保持精简，核心规则控制在 200 行以内。用"只读必要文件"原则管理细节。

**Q: 多个 Agent 角色怎么写？**
A: 用 `## 角色：{role}` 分节，或拆成多个文件放在 `.claude/agents/` 目录。

**Q: 规则冲突时谁优先？**
A: 禁止事项 > 工作规则 > 默认行为。明确写出优先级。
